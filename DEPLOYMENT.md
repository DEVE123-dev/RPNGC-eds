# 🚀 RPNGC Dispatch - Deployment Guide

## Quick Deploy Options

### Option 1: Render (RECOMMENDED - FREE & EASY)

1. **Push to GitHub** (if not already)
   ```bash
   git init
   git add .
   git commit -m "Ready for deployment"
   git remote add origin YOUR_GITHUB_REPO_URL
   git push -u origin main
   ```

2. **Deploy on Render**
   - Go to https://render.com
   - Click "New +" → "Web Service"
   - Connect your GitHub repo
   - Render auto-detects `render.yaml`
   - Add environment variables:
     - `MONGO_URI`: Your MongoDB connection string
     - `JWT_SECRET`: Auto-generated (or set your own)
   - Click "Create Web Service"
   - **Done!** Your app will be live in ~2 minutes

**URL**: `https://rpngc-dispatch.onrender.com`

---

### Option 2: Railway (FAST & SIMPLE)

1. **Push to GitHub** (if not already)

2. **Deploy on Railway**
   - Go to https://railway.app
   - Click "New Project" → "Deploy from GitHub repo"
   - Select your repo
   - Add environment variables:
     ```
     MONGO_URI=your_mongodb_connection_string
     JWT_SECRET=your_secret_key_here
     PORT=8000
     NODE_ENV=production
     ```
   - Railway auto-deploys
   - **Done!**

**URL**: Auto-generated by Railway

---

### Option 3: Heroku (CLASSIC)

1. **Install Heroku CLI**
   ```bash
   # Windows
   winget install Heroku.HerokuCLI
   ```

2. **Login & Create App**
   ```bash
   heroku login
   heroku create rpngc-dispatch
   ```

3. **Set Environment Variables**
   ```bash
   heroku config:set MONGO_URI="your_mongodb_connection_string"
   heroku config:set JWT_SECRET="your_secret_key_here"
   heroku config:set NODE_ENV=production
   ```

4. **Deploy**
   ```bash
   git push heroku main
   heroku open
   ```

**URL**: `https://rpngc-dispatch.herokuapp.com`

---

### Option 4: Vercel (SERVERLESS)

   npm i -g vercel

2. **Deploy**

**URL**: Auto-generated by Vercel

---

### Option 5: Docker (ANY PLATFORM)

1. **Build Image**
   ```bash
   docker build -t rpngc-dispatch .
   ```

2. **Run Container**
   ```bash
   docker run -d -p 8000:8000 \
     -e MONGO_URI="your_mongodb_connection_string" \
     -e JWT_SECRET="your_secret_key" \
     -e NODE_ENV=production \
     --name rpngc-dispatch \
     rpngc-dispatch
   ```

3. **Or use Docker Compose**
   ```bash
   # Create .env file first with your variables
   docker-compose up -d
   ```

**URL**: `http://localhost:8000` or your server IP

## Pre-Deployment Checklist

### 1. Secure Your Environment Variables

**CRITICAL**: Never commit `.env` to Git!

Create `.env` file locally:
```env
MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/rpngc
JWT_SECRET=use-a-strong-random-32-character-secret-key
PORT=8000
NODE_ENV=production
```

Generate strong JWT secret:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 2. MongoDB Atlas Setup

1. Go to https://cloud.mongodb.com
2. Create free cluster (if not exists)
3. **Network Access**: Add `0.0.0.0/0` (allow all IPs)
4. **Database Access**: Create user with read/write permissions
5. Get connection string from "Connect" → "Connect your application"
6. Replace `<username>`, `<password>`, and database name

### 3. Seed Database (First Time Only)

```bash
node db/seed.js
```

This creates:
- 3 sample officers (RPNGC-001, RPNGC-002, RPNGC-003)
- 1 HQ admin (hq-admin)
- Default passwords: `password123` and `hqpassword123`

**⚠️ CHANGE THESE PASSWORDS IN PRODUCTION!**

### 4. Test Locally Before Deploy

```bash
npm start
```

Visit `http://localhost:8000` and test:
- ✅ Citizen portal loads
- ✅ Officer login works
- ✅ HQ login works
- ✅ Map displays correctly
- ✅ Real-time updates work

## Troubleshooting

### App Crashes on Startup

**Check logs for**:
- MongoDB connection errors → Verify `MONGO_URI`
- Missing environment variables → Add in platform dashboard
- Port binding issues → Let platform auto-assign `PORT`

### Socket.io Not Working

- Ensure platform supports WebSockets (Render, Railway, Heroku do)
- Check firewall/proxy settings
- Verify HTTPS is enabled (required for geolocation)

### Database Connection Timeout

- MongoDB Atlas: Whitelist `0.0.0.0/0` in Network Access
- Check connection string format
- Verify database user has correct permissions

### Map Not Loading

- Check browser console for errors
- Verify HTTPS is enabled (geolocation requires secure context)
- Test internet connectivity

## Performance Tips

### For Production

1. **Enable Compression**
   ```bash
   npm install compression
   ```
   Add to `app.js`:
   ```javascript
   const compression = require('compression');
   app.use(compression());
   ```

2. **Add Rate Limiting**
   ```bash
   npm install express-rate-limit
   ```

3. **Use PM2 for Process Management** (VPS/Docker)
   ```bash
   npm install -g pm2
   pm2 start app.js -i max
   ```

4. **Database Indexing**
   Already configured in `data-model.js`:
   - Geospatial index on officer locations
   - Index on userId, badgeNumber, status
